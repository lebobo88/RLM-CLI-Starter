# ADR-001: CLI Framework Selection
## Status: Accepted
## Date: 2026-02-14

## Context
The project requires a robust Command Line Interface (CLI) framework to handle argument parsing, command structure, help generation, and plugin extensibility. We considered `commander`, `yargs`, and `oclif`.

## Decision
We will use **oclif** (Open CLI Framework).

## Consequences
- **Pros**:
    - Modular architecture with separate command files.
    - Built-in support for TypeScript.
    - Strong testing utilities.
    - Auto-generated documentation (README/Help).
    - Plugin system allows for future extensibility.
- **Cons**:
    - Slightly steeper learning curve than `commander` for very simple CLIs.
    - More boilerplate code.
